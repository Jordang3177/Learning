  • Breaking a big project into smaller pieces can help because it allows for you to understand what is needed for you to do at each step. And it is mainly important because then you can test your code at each step and then you can see if it works or doesn't work rather than doing it all at once and not understanding what to do when you did it all at once and it fails. Then it's hard to see what it wrong with your code unlike if you did it piece by piece and tested it at each point.
  • We want to keep checking and testing our code because then we can see where we went wrong as soon as possible rather than after we did alot of coding and realize that it's still broken. This makes it easier to debug code that is constantly checked on.
